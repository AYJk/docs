# 外出探索 + 阶段任务队列 设计 v0.1 —《回收站怪才》

> 配套文档：`核心循环设计_v0.1.md`、`配方与数值设计_v0.1.md`、`项目进展.md`
> 本文是「下一个冲刺」的落地范围设计，供决策评审。**确认后再动代码。**
> 一句话：用「外出探索」替换原设计的「NPC 上门卖废品」作为**在线主材料源**，配「废料包/背包」解决格子溢出，配「阶段任务队列」做目标层——三者合起来正面解决"合成完就没事做"的节奏问题。

---

## 0. 背景与定位（为什么是这个）

### 0.1 两处设计调整（相对既有文档）
- **材料来源改向**：原 `核心循环设计 §四` 把「熟人面具 NPC 上门卖废品」当在线材料源。本次**用「外出探索」替换**它——主角主动出门淘货，更贴「回收站怪才」主角幻想，且自带"爆发式补给 + 随机事件"，比被动等 NPC 敲门更有节奏和惊喜。
- **NPC 不废弃，降级为事件**：上门收购的人设（神秘收藏家等）改为**探索随机事件**里出现（见 §2.6），保留经营味；而**已做好的「订单委托队列」（NPC 买成品）原样保留**，本次只把它从"线性"升级为"阶段并行"（见 §3）。

### 0.2 要解决的真问题：合成节奏（"很快没事做"）
拆成两种，分别对症：
- **横向（会话之间，明天为啥回来）** → 阶段任务队列（目标层）+ 已有的离线收益。
- **纵向（一局之内，合完就干等）** → **靠材料经济从"滴灌"改"爆发"**。现状自动产出 Lv1=2 个/分钟（30s 挤一个），等于大部分时间在等料。外出探索一趟带回一批"废料包"，把滴灌变成「出门一趟 → 回来痛快合一阵 → 见底 → 再出门」。
- **关键认知**：F2P 合成游戏里"料耗尽、暂时没事做"**不是 bug，是设计好的会话边界**——目标是"5~10 分钟爽快爆发 → 干净地停 → 想回来（或看广告续）"，而不是无限有事做。

## 1. 核心循环（接入后）

```
   ┌─ 阶段任务队列（目标层：本阶段要做哪些成品，可乱序，全做完解锁下一场景）
   │
   ▼
 外出探索（燃料层）──带回──> 废料包 → 背包（缓冲，有上限）
   │  · 后台并行计时，可看广告加速                 │
   │  · 完成触发随机事件（惊喜/稀有/小风险）        │ 拖到棋盘空格、点击拆包
   │                                              ▼
   └────────────────────────────────> 棋盘：第一层升级 → 第二层改造工位（倒计时）→ 成品
                                                                    │
                                          成品库存 ──交付──> 任务/订单（兑现：金币 + 口碑）
```

**为什么不会卡壳**：永远有目标可奔（任务队列）、永远能派下一趟探索（燃料）、加工/交付并行推进；真见底了就是该走的会话边界（弹"今日探索已尽 / 看广告再探一次"）。

## 2. 外出探索

### 2.1 流程
1. 点「外出探索」→ 选一个**探索点** → 开始（后台计时，墙钟驱动，复用工位那套 `endAt/savedAt`）。
2. 计时中可继续玩棋盘；**可看广告/花✦立即返回**（MVP 先免费占位）。
3. 返回时：发放**收获 = N 个废料包**，并 roll 一次**随机事件**（§2.6）。
4. 废料包进**背包**（棋盘外，有容量上限）。
5. 棋盘有空格时，从背包**拆包**：每次吐 1 个料进空格 → 合成腾地方 → 再拆。**棋盘满则拆不出**（这是节奏阀，不是 bug）。

> **节流靠三件，不引入体力系统**：探索位数量（MVP=1）+ 单趟计时 + 背包上限。时间就是货币，MVP 探索**免费**（不耗软币）。

### 2.2 探索点（MVP 2 个，后续扩）

| 探索点 | 解锁 | 单趟耗时 | 收获(废料包) | 掉落池 | 事件 |
|--------|------|---------:|-------------:|--------|------|
| 小区垃圾站（近） | 开局 | 60s | 2 包 | 已解锁系的 Lv1（复用 `getSpawnPool`） | 基础事件表 |
| 城郊废墟（远） | 口碑≥30 或 回收站 Lv3 | 600s(10min) | 3 包 | Lv1 为主 + 少量 Lv2；**事件可出 Lv3/Lv4 稀有** | 含稀有箱/收藏家 |

- **近点**=日常工作马：短、随时、量小，撑住"出门-回来"的高频节奏。
- **远点**=挂机/期待：长、量大，且是**稀有材料（Lv4）唯一入口**——点亮一直空转的 `RARITY_MULTIPLIER` / 分档 `CRAFT_TIME` 和 67 个未实装高稀有配方。
- **首次探索即时返回**（教学，不让冷启动干等）。

### 2.3 收获模型 = 废料包（解决"一趟把格子塞满"）
- 探索**不发散料，发"废料包"**。料是**潜能**，不是实物——只在你拆、且有空格时才落地。
- 一个废料包 = 若干「电量(charges)」：每点 1 次吐 1 个料。**棋盘满 → 吐不出**，逼你先合一波。
- 这样**溢出从根上不存在**：出料速度由玩家 × 棋盘空间共同决定。一趟的"潜能"够你 working through 一整局，burst 拉成持续曲线。

### 2.4 背包（缓冲区）
- 棋盘外的一行「废料包槽」，**容量上限**（MVP=6 包，后续可升级/付费扩）。
- 探索回来包先进背包；**背包满则不能再探**（"先处理掉再出门"的压力，也防无限囤积）。
- 升级背包容量 = 后续一个软币/付费杠杆（先不做）。

### 2.5 拆包交互（MVP）
- 背包里点一个废料包 → **吐 1 个料到棋盘任一空格**（无空格则提示"先腾出空间"）。包上显示剩余电量，吐空自动消失。
- 取**1 点 1 个**（克制、保留棋盘决策），不取"一键全倒"（会瞬间洪水）。电量给足 + 出料干脆，手感是"拆一包货"而非"抠搜微操"。
- 〔白盒实现〕废料包 = 一个色块 + 数字角标；点击即吐，复用 `gridManager.addItem` 落空格。

### 2.6 随机事件（初版表，权重待调）
返回时 roll 一次。事件 = 一段文案 + 至多一个小抉择，**内容廉价、AI 易扩**，是给"重复合成"解腻的关键。

| 事件 | 权重 | 效果 | 备注 |
|------|---:|------|------|
| 普通收获 | 55% | 标准收获，无事件 | 基线 |
| 满载而归 | 12% | 本趟废料包 +50% | 正反馈 |
| 锈箱子 🎁 | 10% | 看广告/花✦ 开 → 1 个 Lv3/Lv4 稀有料（可放弃） | **稀有入口 + 广告钩子** |
| 神秘收藏家 | 6% | 拿 X 个普通料换 1 个稀有料（保留 NPC 味） | 收编原"上门 NPC" |
| 遇到老主顾 | 6% | 口碑 +N（或额外冒一个任务） | 接任务队列 |
| 拾金不昧 | 6% | +少量金币 | 调味 |
| 空手而归/被赶 | 5% | 收获 -50%（小风险） | 制造起伏，非惩罚性 |

> 远点的事件表把"锈箱子/收藏家"权重调高；近点几乎只出基线 + 小正反馈。

### 2.7 加速 / 离线 / 防作弊
- **加速**：看广告或花✦ 立即返回（MVP 免费占位，留 SDK 接口；数值参照 `改造时间表` 的✦档位）。
- **离线**：探索计时与工位一致——离线期间趟次照常完成，回来在**回归弹窗**里一并汇总（"离线 Xh，探索带回 N 包 / 工位完成 M 个"）。
- **防作弊**：同 `项目进展 §7-9` 已知限制——往前调设备时间可瞬间完成趟次，本地只做回拨检测 + 封顶兜底，根治待后端校时。**不在本次重复造。**

## 3. 阶段任务队列（升级既有订单队列，非新系统）

### 3.1 现状 → 目标
- **现状**：`GameConfig.NPC_ORDER_QUEUE` 6 条，`orderIndex++` **线性**逐个解锁，必须按顺序。
- **目标**：把**一个阶段（=一个场景）的全部任务一次摊开**，**可乱序**完成；**全部完成 → 解锁并切换下一场景**，露出下一阶段任务。
- **本质**：本游戏里"任务"就是"订单"，二者是同一系统——这是把它从线性改成**阶段并行**，不是另起炉灶。

### 3.2 数据结构调整
- 给订单按 `sceneId` 归组为"阶段"。现有 6 条全是 `garage` → 即第一阶段（车库）。
- 运行态：`orderIndex:number` → `completedOrders:Set<id>`（记已完成，判断阶段是否清空）。
- 阶段清空 → 触发 `unlockSceneId` 切场景。`order_006`（搬店大单）可保留为车库阶段的一条"大单"，与其余 5 条并列；6 条全交付 = 车库阶段通关 → 解锁街边小店。

### 3.3 场景解锁
- 车库（garage）阶段：6 条并行任务（沿用现有文案/奖励/口碑）。
- 街边小店（street_shop）阶段：**目前无任务内容**——MVP 可先"解锁但显示'装修中，敬请期待'"，或补 2~3 条新委托（内容待产）。
- 场景切换的视觉反馈（搬迁动画/换底图）后置，先做状态切换 + 文案。

## 4. 与现有系统的接合

| 现有 | 处理 |
|------|------|
| **自动产出**（2/分钟滴灌） | 退为**离线/挂机被动源**（继续喂已做好的离线收益）；在线下相对探索可忽略。是否在线时干脆关掉、只留离线 = 一个调参点。 |
| **回收按钮** | **替换为「外出探索」入口**（避免两个在线主动源打架）。 |
| **改造工位 / 第一层合成** | 不动，照常。探索只负责把料送上棋盘。 |
| **存档** | v2 → **v3**：新增探索趟次状态、背包废料包、探索点解锁、`completedOrders`。旧档缺省值兼容（同订单队列那次做法）。 |
| **回归弹窗** | 汇总项加"探索带回 N 包"。 |
| **墙钟/防作弊** | 探索复用工位的 `endAt/savedAt` 墙钟方案，已知时间作弊限制照旧。 |

## 5. 数值初值表（可直接落 GameConfig）

| 项 | 初值 | 说明 |
|----|------|------|
| 探索位数量 | 1 | 后续回收站升级解锁第 2 个 |
| 近点耗时 / 收获 | 60s / 2 包 | 高频工作马 |
| 远点耗时 / 收获 | 600s / 3 包 | 稀有入口；口碑≥30 或 Lv3 解锁 |
| 废料包电量 | 近 6 / 远 8 | 一趟 ≈ 12~24 个潜在料（16 格棋盘吃不下，必须边合边拆） |
| 背包上限 | 6 包 | 满则不能再探 |
| 拆包 | 1 点 1 个料，棋盘满则阻塞 | — |
| 首次探索 | 即时返回 | 教学 |
| 稀有箱掉落 | Lv3 或 Lv4，按掉落池 | 远点专属 |
| 加速 | MVP 免费占位 | 留广告/✦ 接口 |

> 自洽校验：Lv1 棋盘 16 格 + 开局 8 料；近趟 60s 带 2 包（≈12 潜在料）→ 边拆边合可支撑数分钟活动；背包封 6 包防囤积。节奏 = 短趟可连发形成 burst，背包上限收口。

## 6. 代码改动面（评估工作量用，确认后才写）

- **GameConfig.ts**：新增 `EXPLORE_SITES`（id/名/耗时/包数/电量/掉落池/解锁条件）、`EXPLORE_EVENTS`（事件表+权重）、`BACKPACK_CAP`；订单按场景归组（或加 `STAGES` 视图）。
- **新增 `core/ExploreSystem.ts`**：镜像 `CraftSystem` 的墙钟模式（`startExplore/updateExplores/collectExplore/speedUpExplore` + `endAt/savedAt` + `restore`），产出"废料包"列表。
- **背包数据**：`GridManager` 扩展或新增轻量持有者，存废料包 + 拆包出料到空格。
- **QuickStart.ts**（主线，改动最大）：
  - 探索 UI：入口按钮（替换回收按钮）+ 探索点选择 + 进行中状态 + 加速 + 背包条 + 拆包交互 + 事件弹窗。
  - 订单面板：线性 → **多任务并行板**（阶段内全列出，乱序交付，全清解锁场景）。
  - `save/load` v3；回归弹窗加探索汇总。
- **量级提示**：这是迄今**单次最大的一块新增**（比订单队列大），含一个新系统 + 一套新 UI 交互 + 存档迁移。建议分两批落（见 §9）。

## 7. MVP 圈定

**做**：近/远 2 个探索点 · 1 个探索位 · 废料包→背包→点击拆包（含满格阻塞）· 6 条基础事件（含 1 稀有箱 + 1 收藏家 = 稀有入口）· 探索加速（免费占位）· 离线趟次并入回归弹窗 · 订单线性→阶段并行（车库 6 条）+ 全清解锁街边小店 · 回收按钮→探索入口 · 存档 v3。

**不做（后续）**：第 2/3 探索位与探索点 · ✦双货币与真广告 SDK · 街边小店专属任务/掉落 · 背包扩容付费 · 每日配方挑战 · 场景切换动画。

## 8. 待你拍板

1. **探索形态**：后台并行（派出去照玩棋盘，推荐——节奏最好） vs 前台等待（盯着进度条）。
2. **拆包手感**：1 点 1 个（克制，推荐） vs 一键全倒（快但易洪水）。
3. **回收按钮**：直接替换成探索入口（推荐） vs 保留为即时小补给。
4. **在线自动产出**：保留小滴灌 vs 只留离线（在线全交给探索）。
5. **稀有箱开启**：MVP 免费占位（推荐，先验玩法） vs 直接挂软币收购价（`料VP×0.4`）。

## 9. 落地顺序建议（若方向通过）

- **批 1（探索燃料闭环）**：ExploreSystem + 探索点 + 废料包/背包 + 拆包 + 替换回收按钮 + 存档 v3。先把"出门→带回→拆→合"跑通，网页预览端到端验证。
- **批 2（目标层 + 惊喜层）**：订单线性→阶段并行 + 全清解锁街边小店；随机事件表 + 稀有箱（点亮稀有度机制）。
- **批 3（验证后）**：远点稀有掉落调参 + 离线趟次回归弹窗 + 真机冒烟（顺带还触控/胶囊验证债）。
